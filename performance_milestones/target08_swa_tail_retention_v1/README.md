## Result

Decision: **reject runtime V1 and prepare TARGET 08.21**.

The new opt-in is present but fail-closed:

```text
--enable-dsv4-swa-tail-retention-v1
SchedulerConfig.enable_dsv4_swa_tail_retention_v1
```

When requested for DeepSeek V4, scheduler construction raises before runtime
cache use.  This is deliberate.  Mini currently derives C4, C128, indexer, and
compression-state locations from full-token pages.  Releasing old full/SWA pages
while retaining those derived components would risk stale reads, component slot
reuse collisions, or double free.  The default path and the phase-1 rollback
path `--enable-dsv4-radix-prefix-cache` remain unchanged.

## SGLang Parity

SGLang has separate full/SWA allocators, a full-to-SWA mapping,
`free_swa_out_of_window_slots()`, an `SWAComponent` that tombstones old SWA, and
`UnifiedRadixCache` match validators that find a component-safe fixed point.
Mini phase-1 does not have those structural pieces: the page table remains the
full-token namespace, and compressed/indexer/state locs are derived from it.

## Mini V1 Data Model

V1 data model: fail-closed safe subset.

No independent component store or component refcount is enabled in runtime.  The
opt-in name is `enable_dsv4_swa_tail_retention_v1`; the CLI spelling is
`--enable-dsv4-swa-tail-retention-v1`.

## Ownership And Free Design

Current safe owner remains the full-token page.  Since V1 does not detach
components, there is no new free path and no new double-free surface.  The guard
exists because a real implementation must first introduce either:

- retained C4/C128/indexer storage plus materialize/remap on hit; or
- independent component allocators/refcounts and attention metadata that can
  address component locations independently of `full_loc // ratio`.

Compression state is not independently owned in mini today; its state loc is
derived from full/SWA page locs.

## Graph Replay

Phase-1 graph rollback reference remained enabled in earlier smoke/perf runs.
V1 `[1,2,4,8,16]` replay/eager counts were not collected because the target
stopped before runtime retention.  The new flag does not alter graph bucket
selection unless explicitly requested, and explicit request fails closed.

## Decision

Reject TARGET 08.20 runtime V1.  Prepare TARGET 08.21 around one of the two
real ownership models:

- retained component store with hit-time materialization/remap and radix
  logical-versus-live accounting; or
- independent component allocators/refcounts plus metadata/kernel support for
  component locs that are not derived from reused full pages.
